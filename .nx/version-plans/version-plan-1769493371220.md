## Breaking Changes

- **Removed `nx-serverless` package**: The deprecated `@aligent/nx-serverless` package has been completely removed from the monorepo. All related generators, plugins, and documentation have been deleted.

## Features

### nx-cdk

- Added bundle mode support for Rolldown bundler in generated applications
- Enhanced package.json template generation with improved dependency management
- Updated NX configuration helpers to support NX 22.4

### Repository

- **New Package Generator**: Added `tools/generators/package` generator for scaffolding new packages in the monorepo
  - Includes TypeScript configuration templates
  - Pre-configured with ESLint and Vitest
  - Generates proper project.json with build and test targets

## Improvements

### All Packages

- **NX Migration**: Upgraded from NX 20.8.1 to 22.4.1
- **Test Configuration**: Migrated from `vite.config.mjs` to `vitest.config.mjs` for better test configuration separation
- **TypeScript Configuration**: Cleaned up and standardized tsconfig files across all packages
  - Removed redundant path mappings from tsconfig.base.json
  - Added root-level tsconfig.json for better IDE support
  - Improved tsconfig.lib.json and tsconfig.spec.json configurations

### Dependencies

- Upgraded `@aligent/ts-code-standards` from 3.0.0 to 4.2.0
- Upgraded `vite` from 6.4.1 to 7.3.1
- Upgraded `vitest` from 3.1.2 to 4.0.9
- Added `@nx/vitest` plugin (22.4.1) to replace `@nx/vite` for test running
- Updated Node.js version to use `.nvmrc`

### microservice-util-lib

- Minor code improvements in retry middleware
- Enhanced remap utility with better type safety

## Maintenance

- Updated .gitignore patterns
- Improved .prettierignore configuration
- Updated README documentation
