### New: @aligent/aws-wrappers package

- Introduces seven AWS SDK service wrappers (`S3Service`, `DynamoDBService`, `SecretsManagerService`, `StepFunctionsService`, `SSMService`, `SQSService`, `SNSService`) that bundle Powertools `Logger` and X-Ray tracing by default.
- Constructor signature is uniform across services: `new XService({ logger?, client? })`. Defaults instantiate a Powertools logger named after the class and wrap the SDK client with `captureAWSv3Client`.
- Every public method emits a structured `logger.info('<verb> <noun>', { input })` line at the start of the call. Batch and write methods log entry counts / `Bucket`+`Key` rather than full payloads to avoid spilling large or sensitive content into CloudWatch.
- Smooths over recurring SDK quirks:
    - **Auto-pagination (flat array)**: `S3.listObjects` / `getAllObjects` / `emptyBucket`, `SSM.getParametersByPath`, `SFN.listExecutions`.
    - **Generator pagination (yields items)**: `DynamoDB.paginateItems` / `paginateScan` — bounded memory for potentially unbounded result sets.
    - **Auto-chunking**: `S3.deleteObjects` / `emptyBucket` (1000-key cap), `SQS.sendMessageBatch` / `deleteMessageBatch` (10-entry cap), `SNS.publishBatch` (10-entry cap).
    - **`DynamoDB.batchWrite` retries `UnprocessedItems`** with jittered exponential backoff (5 attempts, 200ms base), throwing if items remain after the final attempt.
- `DynamoDBService` is backed by `DynamoDBDocumentClient`; X-Ray wraps the base client *before* `DocumentClient.from` so every command is traced. Generics default to `T = Record<string, unknown>`. Methods that return command output (`query`, `scan`, `updateItem`, `deleteItem`) preserve the full output and type only the data-bearing field (`Items?: T[]`, `Attributes?: T`). `batchGet` is intentionally non-generic — multi-table `Responses` cannot be soundly described by a single `T`.
- `SSMService.getParameters` takes an alias-to-path record so callers destructure with the names they want to use locally:
    ```ts
    const { username, password } = await ssm.getParameters({
        username: '/myapp/db/username',
        password: '/myapp/db/password',
    });
    ```
    `WithDecryption: true` is baked into all SSM operations.
- Includes a package-level `CLAUDE.md` capturing the locked-in conventions, design rationale, the question checklists for adding a new service or modifying an existing one, and the `aws-sdk-client-mock` paginator-with-real-client test pattern.
