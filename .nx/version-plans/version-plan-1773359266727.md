nx-cdk: minor

- Add resource prefix aspect support to generated CDK applications, allowing resources to be prefixed based on the stage name
- Support dynamic stage creation in CDK app via context (`-c environment=dev`), replacing the static stage approach
- Set `dev` as the default stage when no environment context is provided
- Separate playground-specific CDK commands from standard commands in generated workspace scripts
- Update `pg:synth`, `pg:deploy`, and `pg:destroy` workspace scripts to adapt with dynamic stage
- Add `environment-target` input to GitHub Actions workflow templates for staging and production deployments
- Update README files for more details on synthesize, deploy, and destroy commands including custom stage examples
