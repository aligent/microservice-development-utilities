## Breaking Changes

**BREAKING:** The generator no longer creates `project.json` configuration files. The generator now uses TypeScript project references (when using `tsconfig.json` at root) or path mappings (when using `tsconfig.base.json`) instead of Nx project configuration.

### Legacy Project Support

If you have a legacy project that requires `project.json`, you will need to manually create the `clients/project.json` file with the following content:

```json
{
  "name": "clients",
  "root": "clients",
  "projectType": "library",
  "sourceRoot": "clients/src",
  "targets": {},
  "tags": ["clients"]
}
```

Note: This is not the recommended approach. New projects should use TypeScript project references instead

## Other Changes

- Updated `tsconfig.json.template` to extend from `@aligent/ts-code-standards/tsconfigs-extend`
- Removed `tsconfig.lib.json.template` (no longer needed with new structure)
- Improved validation logic: schema validation now runs by default but only throws error if `skipValidate` is false
- Refactored utilities:
  - Replaced `attemptToAddProjectConfiguration` with `getExistingProject` for better semantics
  - Added `addTsConfigReference` function to support TypeScript project references
  - Exported `getRootTsConfigPathInTree` for external use
  - Updated `addTsConfigPath` to accept `tsConfigFile` parameter
- Enhanced code documentation and comments throughout the generator
